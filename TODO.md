# 🚀 Telemetry Platform MVP Checklist

## 🔧 Backend: конфигурация и окружение

- [ ] Вынести конфигурацию в .env  
       → Создать файл `.env` в корне проекта  
       → Добавить: DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD, OWEN_LOGIN, OWEN_PASSWORD

- [ ] Подключить настройки через Pydantic Settings  
       → Создать класс Settings  
       → Загружать значения из .env  
       → Убрать все хардкоды из кода

- [ ] Исправить DATABASE_URL для Docker  
       → Использовать host = `timescaledb` (имя сервиса из docker-compose)  
       → Проверить, что backend подключается к БД внутри контейнера

---

## 🔄 Polling: стабильность и оптимизация

- [ ] Убрать логин при каждом запросе  
       → Сделать переменные token + expire_time  
       → Использовать токен, если он ещё валиден  
       → Логиниться только при отсутствии/просрочке

- [ ] Добавить retry при запросах к внешнему API  
       → Обернуть запрос в цикл (3 попытки)  
       → Между попытками делать sleep (1–2 сек)

- [ ] Добавить обработку ошибок (try/catch)  
       → Обернуть весь polling в try/except  
       → Логировать ошибки, но не падать

- [ ] Добавить логирование  
       → Логировать:
  - старт polling
  - успешное получение данных
  - ошибки  
    → Использовать logging.basicConfig(level=INFO)

---

## 💾 Data Layer: сохранение данных

- [ ] Проверить корректность записи данных в БД  
       → Убедиться, что записи реально создаются  
       → Проверить поля (value, timestamp)

## 🌐 API: получение данных

- [ ] Сделать endpoint /metrics/today-revenue
      → Возвращает одно число: выручка за сегодня
      → Логика: последний value за сегодня – последний value за вчера

- [ ] Проверить формат ответа
      → JSON для фронта:
